# 環 -wa- 予約フォーム — セットアップ手順

## ファイル構成

```
予約フォーム/
├── reserve.html        ← 予約フォーム本体（GitHub Pages で公開）
├── images/
│   ├── coin.jpeg       ← コインリングの写真
│   └── spoon.jpg       ← スプーンリングの写真
├── gas/
│   └── code.gs         ← Google Apps Script
└── README.md           ← このファイル
```

---

## 1. スプレッドシートの確認

`gas/code.gs` の先頭に既存スプレッドシートの ID が設定されています。

```javascript
var SPREADSHEET_ID = '133OkFa0P-2zrO1ESU93YpGGwf_1rwtHzeT9NcVsrakM';
```

別のスプレッドシートを使う場合は、URL の太字部分に差し替えてください：  
`https://docs.google.com/spreadsheets/d/`**`スプレッドシートID`**`/edit`

---

## 2. GAS プロジェクトの作成

1. 対象のスプレッドシートを開き、メニューから  
   **「拡張機能」→「Apps Script」** を開く

2. 左側ファイル欄の `コード.gs` を選択し、既存の内容を**すべて削除**する

3. `gas/code.gs` の内容をコピー＆ペーストし、**「保存」** をクリックする

---

## 3. Web アプリとしてデプロイ

1. 右上の **「デプロイ」→「新しいデプロイ」** をクリック

2. 歯車アイコン →「ウェブアプリ」を選択

3. 以下の通りに設定する

   | 項目 | 設定値 |
   |------|--------|
   | 説明 | `環 -wa- 予約フォーム v1` |
   | 次のユーザーとして実行 | **自分（自分のGoogleアカウント）** |
   | アクセスできるユーザー | **全員** |

4. **「デプロイ」** をクリックし、アクセス許可ダイアログが出たら承認する  
   ※「このアプリは確認されていません」と表示された場合は  
   「詳細」→「〜（安全ではないページ）に移動」をクリック

5. 表示された **ウェブアプリの URL** をコピーする  
   （例：`https://script.google.com/macros/s/AKfycb.../exec`）

---

## 4. reserve.html へ URL を貼り付ける

`reserve.html` の先頭付近の定数を書き換える：

```javascript
// 変更前
const ENDPOINT_URL = 'ここにデプロイ後のURLを貼る';

// 変更後
const ENDPOINT_URL = 'https://script.google.com/macros/s/AKfycb.../exec';
```

---

## 5. GitHub Pages への公開

1. このフォルダの内容を GitHub リポジトリにプッシュする  
   （`reserve.html`・`images/`・`gas/`・`README.md` をルートに配置）

2. リポジトリの **「Settings」→「Pages」** を開く

3. **「Source」** で `main` ブランチのルート（`/（root）`）を選択し保存する

4. 数分後に以下の URL で公開される  
   `https://<ユーザー名>.github.io/<リポジトリ名>/reserve.html`

---

## 動作確認

1. `reserve.html` をブラウザで開き、フォームを最後まで操作して送信する

2. スプレッドシートを開き **「予約」シート** に行が追加されているか確認する

3. `ENDPOINT_URL` が未設定（デフォルト値のまま）の場合は **デモモード** で動作し、  
   実際の送信は行われず完了画面が表示される（開発中のテストに便利）

---

## GAS を更新した場合（コードを修正したとき）

**新しいデプロイではなく「デプロイを管理」で更新**すること。  
新規デプロイすると URL が変わり、`reserve.html` 側の書き換えも必要になる。

1. 「デプロイ」→「デプロイを管理」を開く
2. 既存デプロイの鉛筆アイコン（編集）をクリック
3. バージョンを「**新しいバージョン**」に変更して「デプロイ」

## スプレッドシートのカラム構成

| 列 | 内容 |
|----|------|
| A  | 予約受付日時 |
| B  | 体験日付 |
| C  | 時間帯（例：13:00-14:00） |
| D  | 体験メニュー（スプーンリング / コインリング） |
| E  | 人数（大人） |
| F  | 子供の人数 |
| G  | お名前 |
| H  | 電話番号 |
| I  | メールアドレス |
| J  | ご要望・備考 |
| K  | ステータス（デフォルト：未確認） |

## 定休日・特定休業日の変更方法

`reserve.html` 内の定数を編集するだけで変更できる：

```javascript
// 定休日の曜日（0=日 1=月 2=火 3=水 4=木 5=金 6=土）
const CLOSED_WEEKDAYS = [1, 2];  // 月・火曜日

// 特定の休業日（例：お盆・年末年始）
const CLOSED_DATES = ['2026-08-13', '2026-08-14', '2026-12-31'];
```

## 予約済み枠の管理方法

現フェーズでは全枠「予約可」表示。  
実際の予約状況を反映したい場合は `reserve.html` 内の定数を編集する：

```javascript
// キー: 'YYYY-MM-DD', 値: 予約済み開始時刻の配列
// 指定した時刻とその直後の30分枠が選択不可になる
const BOOKED_SLOTS = {
  '2026-05-25': ['13:00', '15:00'],
};
```

将来的にはこの定数を GAS の API エンドポイントに差し替えることで  
リアルタイム空き確認が実現できる構造になっている。
